Universal Memory Layer · v2.0.0

ping-mem

Persistent, intelligent, contextually-aware memory for AI agents across sessions, tools, and applications. Acts as shared infrastructure — any agent or app reads/writes through one consistent layer.

Bun + TypeScript SQLite · Neo4j · Qdrant MCP + REST + HTMX REST :3003 SSE/MCP :3003 76+ audit findings resolved 11 Web UI routes
Capabilities

Feature Map

01
Persistent Memory
Key/value memory with metadata (category, priority, tags). Survives across sessions. Full CRUD via MCP tools or REST API. FTS + fuzzy search.
02
Session Management
Isolated sessions per agent or conversation. Start/end lifecycle, session-scoped memory, cross-session queries. Checkpoint and restore for crash recovery.
03
Event Sourcing
Immutable append-only EventStore logs every mutation (SESSION_STARTED, CONTEXT_SAVED, CHECKPOINT_CREATED, etc.). Full audit trail with replay capability.
04
Code Ingestion
Deterministic codebase understanding — Merkle-tree scanning, AST-aware chunking, git history extraction. Produces temporal code graph (Neo4j) + vector index (Qdrant).
05
Semantic Code Search
Search ingested code by natural language query. Hash-based deterministic vectors (no ML required) or optional OpenAI/Gemini embeddings. Filter by project, file, chunk type.
06
Temporal Code Graph
Neo4j bi-temporal model — query files at a point in time, trace file history, walk commit timeline. Nodes: Project, File, Chunk, Commit. Explicit "why" from commit messages.
07
Multi-Agent Orchestration
Register agents with configurable TTL. Per-agent byte and count quotas enforced at write time. Memory isolation by agentId. Admin can list, revoke, and manage agents.
08
Knowledge Store
Structured knowledge entries (title, solution, symptoms, root cause, tags). Cross-project FTS5 full-text search. Upsert by projectId + title. Surfaces institutional learnings.
09
Memory PubSub
Real-time event bus — MEMORY_CHANGED, SESSION events published to subscribers. SSE stream at /api/v1/events/stream. Powers live Web UI updates.
10
Diagnostics & SARIF
Ingest SARIF 2.1.0 from TypeScript, ESLint, Prettier. Content-addressable analysis IDs. Diff two runs, compare tools, group by symbol. Optional LLM summaries.
11
Memory Compression
Compact older memory items into digest facts. Heuristic mode (no LLM) or LLM-powered summarization. Keeps context windows lean while preserving key decisions.
12
Web UI + Admin
10 HTMX server-rendered routes — dashboard, memories, agents, knowledge, sessions, events, diagnostics, ingestion, worklog. Admin panel with API key management (AES-256-GCM).

Path-Independent Project IDs

ProjectId = SHA-256(remoteUrl + "::" + relativeToGitRoot). Same project produces the same ID on any machine, in any Docker container, at any path. Docker maps /Users/umasankr/Projects → /projects.

Ingestion Requires Neo4j + Qdrant

Core memory (CRUD, sessions) works with SQLite only. Code ingestion endpoints return 503 if Neo4j/Qdrant are not running. Stale .ping-mem/manifest.json blocks re-ingestion — delete to force reingest.

Ingestion Pipeline Order

01
IngestionOrchestrator
Scan + chunk + extract git history. Shows progress N/M.
02
TemporalCodeGraph
Persist to Neo4j. Slowest phase — sequential commit processing.
03
CodeIndexer
Vector index into Qdrant. Starts only after Neo4j completes.

Code Intelligence

Code Ingestion System (v1.1.0)

PS
ProjectScanner
src/ingest/ProjectScanner.ts
  • Merkle tree hashing for project integrity
  • Content-addressable project/file IDs (SHA-256)
  • Manifest storage at .ping-mem/manifest.json
  • Change detection for incremental updates
  • Path-independent projectId from git remote
CC
CodeChunker
src/ingest/CodeChunker.ts
  • Separates code vs comments vs docstrings
  • TypeScript/JS: // and /* */ comments
  • Python: # and """/''' docstrings
  • Deterministic chunk IDs (SHA-256)
  • Line number tracking for provenance
GH
GitHistoryReader
src/ingest/GitHistoryReader.ts
  • Full commit DAG extraction
  • File change tracking (A/M/D/R/C)
  • Unified diff parsing with hunk ranges
  • Explicit "why" from commit messages only
  • Parses: Why: Reason: Fixes # Closes #
  • Never guesses or infers — explicit sources only
DV
DeterministicVectorizer
src/search/DeterministicVectorizer.ts
  • Hash-based feature vectors — no ML required
  • Bit-for-bit reproducible across runs
  • N-gram generation (1-3 grams)
  • L2 normalization
TCG
TemporalCodeGraph
src/graph/TemporalCodeGraph.ts
  • Neo4j bi-temporal model
  • Nodes: Project, File, Chunk, Commit
  • Edges: HAS_FILE, HAS_CHUNK, MODIFIES, CHANGES
  • Point-in-time queries, file history, commit timeline
CI
CodeIndexer
src/search/CodeIndexer.ts
  • Qdrant vector indexing for code chunks
  • Full provenance metadata per chunk
  • Semantic search with project/file/type filters

REST API Reference

REST Endpoints (port 3003)

Codebase Search is GET, Not POST

GET /api/v1/codebase/search?query=...&projectId=...&type=...&limit=... — use query params, not request body. A common mistake.

Method Endpoint Body / Params Notes
POST /api/v1/codebase/ingest {projectDir, forceReingest?} Triggers full ingestion pipeline
POST /api/v1/codebase/verify {projectDir} Verify manifest integrity
GET /api/v1/codebase/search ?query=&projectId=&type=&limit= Semantic code search — NOT POST
GET /api/v1/codebase/timeline ?projectId=&filePath=&limit= Temporal commit history
GET /health — Always 200 if server is up
POST /api/v1/agents/register {agentId, role, admin?, ttlMs?, quotaBytes?, quotaCount?} Register or update agent
GET /api/v1/agents/quotas ?agentId= (optional) Quota usage
DELETE /api/v1/agents/:agentId — Deregister agent
POST /api/v1/knowledge/ingest {projectId, title, solution, symptoms?, rootCause?, tags?} Upsert knowledge entry
POST /api/v1/knowledge/search {query, projectId?, crossProject?, tags?, limit?} Full-text search (FTS5)
GET /api/v1/events/stream — SSE real-time event stream
GET /ui/* — HTMX web UI routes
GET /admin Basic Auth required Admin panel

Failure Modes & Known Bugs

Runtime Failures

Symptom Cause Fix
503 on codebase endpoints IngestionService not initialized docker-compose restart ping-mem
Empty search results Project not ingested into Qdrant bun run scripts/force-ingest.ts <dir>
Wrong projectId Path mismatch Docker vs local Verify git remote URL is consistent
ECONNREFUSED :3003 ping-mem container down docker-compose up -d ping-mem
ECONNREFUSED :6333 Qdrant down docker restart ping-mem-qdrant
No re-ingest after changes Stale manifest blocks scan Delete .ping-mem/manifest.json, reingest
Large repo ingestion hangs Neo4j sequential commit processing Wait — 1000 commits takes significant time

Security Issues (all resolved)

Severity Issue Status
P0 SQL injection via string interpolation Fixed — parameterized queries
P0 Command injection in commit hashes Fixed — SafeGit with execFile
P0 Timing attack in admin auth Fixed — crypto.timingSafeEqual
P0 Unvalidated JSON in API bodies Fixed — Zod schemas on all endpoints
P1 Neo4j session leaks Fixed — try-finally on all sessions
P1 Race conditions (save + max-session) Fixed — promise-based mutex
P1 CORS too permissive Fixed — strict origin policy
P1 No rate limiting Fixed — per-IP sliding window + Retry-After
P1 Invalid session ID accepted Fixed — session existence check on all context endpoints
P1 Agent memory not isolated Fixed — X-Agent-ID propagated to MemoryManager
